# GoNext — Дневник туриста

Мобильное приложение для планирования поездок и ведения личного дневника путешествий. Работает **полностью офлайн**, все данные хранятся локально на устройстве.

- **Платформа:** React Native (Expo)
- **Навигация:** Expo Router
- **UI:** React Native Paper
- **Хранение:** SQLite (expo-sqlite), фотографии — в файловой системе приложения

Подробное описание проекта — в [PROJECT.md](PROJECT.md). План реализации — в [PLAN.md](PLAN.md).

---

## Требования

- **Node.js** 18+
- **npm** или **yarn**
- Для запуска на устройстве: **Expo Go** (Android/iOS) или среда для сборки (Android Studio / Xcode)

---

## Установка и запуск

### 1. Клонирование и установка зависимостей

```powershell
cd c:\Work\GoNext
npm install
```

### 2. Запуск в режиме разработки

```powershell
npm start
```

Откроется Metro bundler и в терминале появятся подсказки:
- **a** — открыть в Android-эмуляторе или на подключённом устройстве (Expo Go)
- **i** — открыть в iOS-симуляторе (только на macOS)
- **w** — открыть в браузере (веб-версия, с ограничениями)

### 3. Запуск на Android (сборка под устройство/эмулятор)

```powershell
npm run android
```

Требуется настроенный Android SDK и при необходимости эмулятор или подключённое устройство с включённой отладкой по USB.

### 4. Запуск на iOS (только на macOS)

```powershell
npm run ios
```

Требуется Xcode и симулятор или подключённое устройство.

### 5. Запуск тестов

```powershell
npm test
```

## Структура проекта

| Каталог / файл | Назначение |
|----------------|------------|
| `app/` | Экраны приложения (Expo Router: файлы = маршруты) |
| `components/` | Переиспользуемые компоненты (карты, фото, галереи) |
| `database/` | Инициализация SQLite и схема БД |
| `services/` | Бизнес-логика: места, поездки, места в поездке, фото, «следующее место» |
| `types/` | TypeScript-типы (Place, Trip, TripPlace, Photo и др.) |
| `utils/` | Утилиты (открытие карты/навигатора) |

Документация по структуре базы данных и API сервисов — в папке [docs/](docs/).

## Основные экраны

- **Главный экран** — кнопки: Места, Поездки, Следующее место, Настройки
- **Места** — список мест, создание/редактирование, карточка места, фото, карта
- **Поездки** — список поездок, создание/редактирование, маршрут (порядок мест, посещения, заметки, фото)
- **Следующее место** — следующее по маршруту текущей поездки место, кнопки «На карте» и «В навигаторе»
- **Настройки** — информация о приложении и экспорт данных

---

## Офлайн и данные

- Все данные (места, поездки, связи, метаданные фото) хранятся в локальной SQLite-базе.
- Фотографии сохраняются в директорию приложения в файловой системе.
- Для работы приложения **интернет не требуется**. Ссылки на карты/навигатор открывают системные приложения (Apple Maps, Google Maps и т.д.); при отсутствии приложения может открыться веб-версия (уже требуется сеть).
